# Доска объявлений (Spring Boot)

Учебный REST-сервис «Доска объявлений», реализованный на Spring Boot. Проект демонстрирует базовые CRUD-операции для сущностей через HTTP-контроллеры.

## Технологии

- Java 17 или выше
- Spring Boot 3.5.5+
- Spring Web
- Maven

## Быстрый старт

### 1. Клонирование репозитория

git clone https://github.com/MatorinFedor/RBPO_2025_demo.git
cd RBPO_2025_demo

http://localhost:8080/api/categories
http://localhost:8080/api/users
http://localhost:8080/api/messages

## Сущности доменной области

Проект включает пять основных сущностей для доски объявлений:

**Listing** — объявление с заголовком, текстом, ценой, категорией и автором.

**Category** — категория объявлений (Техника, Услуги, Недвижимость и т.д.).

**User** — пользователь сервиса, автор объявлений и отправитель сообщений.

**Message** — сообщение к объявлению для обсуждения условий и уточнения деталей.

**Report** — жалоба на объявление или пользователя для системы модерации.

## REST-эндпоинты

На текущем этапе каждая сущность имеет полный набор CRUD-операций. Контроллеры возвращают текстовые ответы без работы с базой данных.

### Объявления (Listing)

`GET /api/listings` — получить все объявления

`GET /api/listings/{id}` — получить объявление по id

`POST /api/listings` — создать объявление

`PUT /api/listings/{id}` — обновить объявление

`DELETE /api/listings/{id}` — удалить объявление

### Категории (Category)

`GET /api/categories` — получить все категории

`GET /api/categories/{id}` — получить категорию по id

`POST /api/categories` — создать категорию

`PUT /api/categories/{id}` — обновить категорию

`DELETE /api/categories/{id}` — удалить категорию

### Пользователи (User)

`GET /api/users` — получить всех пользователей

`GET /api/users/{id}` — получить пользователя по id

`POST /api/users` — создать пользователя

`PUT /api/users/{id}` — обновить пользователя

`DELETE /api/users/{id}` — удалить пользователя

### Сообщения (Message)

`GET /api/messages` — получить все сообщения

`GET /api/messages/{id}` — получить сообщение по id

`POST /api/messages` — создать сообщение

`PUT /api/messages/{id}` — обновить сообщение

`DELETE /api/messages/{id}` — удалить сообщение

### Жалобы (Report)

`GET /api/reports` — получить все жалобы

`GET /api/reports/{id}` — получить жалобу по id

`POST /api/reports` — создать жалобу

`PUT /api/reports/{id}` — обновить жалобу

`DELETE /api/reports/{id}` — удалить жалобу

## Тестирование через Postman

Для проверки работы эндпоинтов используй Postman или аналогичный инструмент:

1. Скачай Postman с https://www.postman.com/downloads/
2. Создай новый запрос и выбери нужный HTTP-метод (GET, POST, PUT, DELETE)
3. Введи URL эндпоинта (например, `http://localhost:8080/api/listings/1`)
4. Нажми Send

Примеры ответов:

GET `http://localhost:8080/api/listings` → "Получить все объявления"

GET `http://localhost:8080/api/listings/1` → "Получить объявление 1"

POST `http://localhost:8080/api/listings` → "Создать объявление"

PUT `http://localhost:8080/api/listings/1` → "Обновить объявление 1"

DELETE `http://localhost:8080/api/listings/1` → "Удалить объявление 1"

## Функции сервиса (план развития)

В дальнейшем на основе текущего REST-скелета планируется реализовать полный функционал доски объявлений:

Размещение объявлений — пользователи могут создавать объявления с заголовком, описанием, ценой, категорией и контактной информацией; редактировать и удалять свои объявления.

Просмотр и поиск — пользователи могут просматривать список всех объявлений, получать информацию о конкретном объявлении, фильтровать по категориям и цене, искать по ключевым словам.

Управление категориями — администратор может создавать и изменять категории для структурирования объявлений.

Управление пользователями — каждый пользователь имеет профиль с информацией об авторе; просмотр собственных объявлений.

Система сообщений — пользователи могут отправлять друг другу сообщения по интересующему объявлению для обсуждения деталей и условий сделки.

Система жалоб и модерации — пользователи могут отправлять жалобы на нарушающие правила объявления и пользователей; модератор может просматривать жалобы, менять их статусы и блокировать нарушителей.
